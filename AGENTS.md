# AGENTS.md

## About this project’s user

The owner of this project is **not a developer**.

This means:
- Keep explanations simple and practical.
- Prefer small, safe changes over large refactors.
- Avoid jargon. If a term is unavoidable, define it briefly.
- When there are multiple options, recommend one default and say why.

## How to work in this repo

- **Be explicit**: describe what you changed and where (file names).
- **Be cautious**: don’t delete things unless clearly requested.
- **Keep it readable**: add short “headline” notes in code so it’s easy to navigate.
- **Minimize surprise**: don’t introduce new libraries or complex patterns unless asked.

## What to do when unsure

- Make the most reasonable, minimal change that matches the request.
- If something looks auto-generated (e.g. Next.js type files), avoid editing it unless necessary.
